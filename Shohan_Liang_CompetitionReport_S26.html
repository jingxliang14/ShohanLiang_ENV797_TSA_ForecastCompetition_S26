We evaluated several forecasting approaches and selected the five best-performing models based on validation RMSE: Seasonal Naive, TBATS, STLF, ARIMAX, and an ensemble model.

Model Fit

SNAIVE

The seasonal naive model was applied to the daily load time series (train_msts) with a 31-day forecast horizon. It generates forecasts by repeating the load values from the same day in the previous weekly cycle, effectively capturing weekly seasonality. This model was evaluated on the July 2010 validation period and used as a baseline to benchmark more complex models.

snaive_fit <- snaive(train_msts, h = 31)

TBATS

STLF

The STLF approach decomposed the load series into seasonal and non-seasonal components using the multi-seasonal structure of train_msts. The seasonally adjusted series was then modeled using exponential smoothing (ETS), and the seasonal component was added back to produce forecasts. A 31-day forecast was evaluated against July 2010 to assess its short-term predictive performance.

STLF_for <- stlf(train_msts, h = 31, method = "ets")

ARIMAX

An ARIMA(0,1,5) model was fitted to the training load series with external regressors. Fourier terms (K = c(3, 10)) were used to represent weekly and annual seasonality, while daily temperature and humidity were included as covariates. Future Fourier terms and observed weather data from the validation period were used to generate a 31-day forecast. This model explicitly incorporated weather effects alongside temporal dynamics.

K_vec <- c(3, 10)

fourier_train <- fourier(train_msts, K = K_vec)

xreg_train <- cbind(
  fourier_train,
  temp = train$daily_temp,
  humi = train$humi_daily
)

ARIMAX_fit <- Arima(
  train$daily_load,
  order = c(0, 1, 5),
  xreg  = xreg_train
)

fourier_val <- fourier(train_msts, K = K_vec, h = 31)

xreg_val <- cbind(
  fourier_val,

ARIMAX_for <- forecast(ARIMAX_fit, xreg = xreg_val, h = 31)

get_metrics <- function(forecast_obj, actual, name) {
  
  # If it's a forecast object
  if ("forecast" %in% class(forecast_obj)) {
    acc <- accuracy(forecast_obj, actual)
    
    tibble(
      model = name,
      RMSE  = acc["Test set", "RMSE"],
      MAE   = acc["Test set", "MAE"],
      MAPE  = acc["Test set", "MAPE"]
    )
    
  } else {
    # If it's a numeric vector (like ensemble)
    tibble(
      model = name,
      RMSE  = sqrt(mean((forecast_obj - actual)^2)),
      MAE   = mean(abs(forecast_obj - actual)),
      MAPE  = mean(abs((forecast_obj - actual) / actual)) * 100
    )
  }
}


model_results <- bind_rows(

  get_metrics(ARIMAX_for,  val$daily_load, "ARIMAX"),
  get_metrics(ensemble_val, val$daily_load, "ENSEMBLE")

Model Comparison

The models were evaluated using RMSE, MAE, and MAPE on the July 2010 validation set.

TBATS achieved the lowest RMSE, indicating it provided the most accurate forecasts among all models. ARIMAX and STLF also performed well, suggesting that both seasonal structure and external covariates contribute to predictive performance. The Seasonal Naive model performed substantially worse, confirming that more sophisticated approaches improve forecast accuracy. The ensemble model did not outperform the best individual model, indicating that averaging did not provide additional gains in this case.
